Costex Parser & Web Dashboard
Цей проєкт — це комплексний багатоетапний парсер (скрапер) для сайту costex.com (та опціонально leiparts.com для додаткових характеристик), написаний на Python з використанням Playwright. Проєкт також включає веб-дашборд на базі Flask для зручного управління процесом парсингу та завантаження результатів.

🌟 Основні можливості
Багатоетапний парсинг (Pipeline): Збір категорій -> збір товарів (через перехоплення AJAX) -> детальний парсинг кожної сторінки товару.

Обхід захисту (Anti-Bot): Використання Playwright з підміною User-Agent, приховуванням webdriver, "людською" поведінкою (затримки, рухи мишкою) та обробкою перевірок Cloudflare.

Резервування та відновлення (Resume): Процес зберігає проміжні результати у stage4_partial.jsonl. Якщо парсер впаде, він продовжить роботу з місця зупинки.

Багатопотоковість: Підтримка декількох потоків для швидшого збору деталей (STAGE4_THREADS).

Веб-дашборд: Локальний веб-інтерфейс для запуску, зупинки, перегляду логів у реальному часі та завантаження фінального .xlsx файлу.

Експорт в Excel: Фінальний результат форматується у стандартну таблицю, готову до імпорту (з колонками Reference, Price, Quantity, Images тощо).

🛠 Технологічний стек
Мова: Python 3.11+

Браузерна автоматизація: Playwright

Веб-сервер (Дашборд): Flask + Waitress

Обробка даних: csv, json, openpyxl, pandas (опціонально)

🚀 Встановлення та налаштування
1. Клонування репозиторію та створення віртуального середовища
Bash
git clone <url_вашого_репо>
cd parser_costex
python -m venv .venv

# Активація (Windows)
.venv\Scripts\activate
# Активація (Linux/Mac)
source .venv/bin/activate
2. Встановлення залежностей
У проєкті є requirements.txt. Проте для роботи дашборду потрібно також встановити flask та waitress.

Bash
pip install -r requirements.txt
pip install flask waitress
3. Встановлення браузерів для Playwright
Bash
playwright install chromium
4. Налаштування змінних середовища (.env)
Створіть файл .env у корені проєкту та додайте наступні змінні:

Ini, TOML
# Авторизація на Costex
COSTEX_LOGIN=your_costex_email@example.com
COSTEX_PASSWORD=your_costex_password

# Налаштування веб-дашборду
FLASK_SECRET_KEY=super-secret-key-123
DASHBOARD_USER=admin
DASHBOARD_PASS=admin123

# Налаштування парсера
DEBUG=False
CONSOLE_MODE=summary # Або 'full' для виводу всіх деталей у консоль
TRACE_NAV=0 # 1 - для логування всіх URL переходів

# Багатопотоковість та відновлення
STAGE4_THREADS=1
RESUME_STAGE4=1 # 1 - продовжувати з останнього місця, 0 - почати спочатку
CLEAR_STAGE4=0

# Опціонально: збір додаткових даних з leiparts.com
USE_LEIPARTS=1
💻 Використання
Проєкт можна запускати двома способами: через графічний веб-інтерфейс або безпосередньо з консолі.

Варіант А: Запуск через Web Dashboard (Рекомендовано)
Запустіть файл сервера:

Bash
python app.py
Відкрийте у браузері http://localhost:5000

Авторизуйтесь (за замовчуванням логін: admin, пароль: admin123 або ті, що ви вказали у .env).

Натисніть ▶ Run Now для старту парсера.

Слідкуйте за логами у реальному часі. Після завершення натисніть 📥 Download Latest XLSX.

Варіант Б: Запуск через консоль (CLI)
Для тестування або запуску на віддаленому сервері (наприклад, через cron):

Bash
python main.py
Усі логи будуть зберігатися у папці logs/, а фінальний результат з'явиться у корені як costex_catalog_latest.xlsx.

🏗 Архітектура Pipeline (main.py)
Парсер розбитий на логічні етапи (Stages), щоб гарантувати стабільність:

Stage 1 & 2 (Категорії та Підкатегорії):

Файл: category.py

Збирає дерево категорій з сайту і зберігає їх у subcategories.csv.

Stage 3 (Збір PART_NO):

Файл: products.py

Заходить у кожну підкатегорію. Оскільки список товарів завантажується динамічно (через AJAX запити від wpDataTables), скрипт перехоплює мережеві запити (page.on("response")) та витягує з JSON всі номери деталей (PART_NO).

Результат: Products_ALL.csv.

Stage 4 (Детальний збір):

Файли: deteil_product.py, authorization.py, parser_features.py

Скрипт логіниться в Costex -> переходить на сторінку "Quote Online" -> вводить кожен PART_NO у форму -> відкриває модальні вікна або сторінки "More Details" -> збирає ціни, вагу, габарити та посилання на фото.

Опціонально заходить на leiparts.com для збору характеристик.

Оформлення результату:

Файл: write.py

Прибирає дублікати, архівує старі файли та створює чистий costex_catalog_latest.xlsx зі строгою структурою колонок.

📂 Структура проєкту
main.py — Головний контролер, який запускає весь Pipeline.

app.py — Flask-додаток (Дашборд).

authorization.py — Логіка "людського" введення логіна/пароля та обробка перевірок після входу.

category.py / products.py — Збір базових посилань та перехоплення AJAX для пошуку товарів.

deteil_product.py — Ядро парсингу характеристик товару. Працює з DOM, чекає появи елементів.

parser_features.py — Окремий парсер для отримання характеристик з Leiparts.

write.py — Експорт у формат Excel (OpenPyXL).

debug_utils.py — Робить скріншоти та зберігає HTML сторінок (dbg/), якщо скрипт падає з помилкою (дуже корисно для налагодження headless-режиму).

config.py — Словники CSS селекторів, User-Agents та налаштування шляхів.

🐛 Налагодження (Debugging)
Якщо парсер падає (особливо на Stage 4), перевірте директорію dbg/ або logs/.
Скрипт автоматично робить скріншот повного екрану (.png) та зберігає .html код сторінки в момент збою.
Це дозволяє зрозуміти, чи змінив Costex верстку, чи з'явилася капча від Cloudflare.
